# 代码评审（Code Reviewer）

本技能用于对提交的代码、差异（diff）或文件进行系统化评审，识别缺陷与风险并给出可操作的改进建议。

## 何时使用

- 当用户明确请求“代码评审/Code Review”
- 代码合并（PR/MR）前的质量门禁
- 需要发现潜在问题、提升可读性/性能/安全性时

## 输入格式

- Git diff/patch
- 文件或目录路径
- 代码片段（多语言）

## 评审维度

- 正确性与鲁棒性：边界条件、异常处理、空值/越界检查
- 安全：注入、越权、敏感信息暴露、输入校验、加密与凭证使用
- 性能：复杂度、热点路径、I/O与网络、内存占用、缓存策略
- 可读性与一致性：命名、风格、模块边界、重复代码、死代码
- 可维护性与架构：职责分离、依赖合理性、扩展性、耦合度
- 测试与文档：单元/集成测试覆盖、断言质量、注释与使用说明

## 输出格式（模板）

- 总览
  - 变更范围与总体风险
- 关键问题（按严重度排序：严重/高/中/低）
  - 文件与行范围
  - 问题描述与影响
  - 修复建议与示例
- 改进建议
  - 可读性/性能/安全/测试/文档等方向
- 审查清单（Checklist）
  - 错误处理是否充分：通过/未通过（说明）
  - 安全输入校验是否到位：通过/未通过（说明）
  - 风格与一致性是否符合项目约定：通过/未通过（说明）
  - 测试是否覆盖关键路径：通过/未通过（说明）
- 结论
  - 是否建议合并：是/否
  - 阻塞项与优先级

## 工作步骤

- 理解上下文：入口、执行路径、依赖与外部接口
- 跟踪数据流与状态变化，识别边界与异常分支
- 检查安全与资源管理（生命周期、并发、泄漏）
- 评估性能与复杂度，提出可行优化方案
- 对齐项目既有风格与约定，减少不必要差异
- 评估测试缺口并建议关键用例
- 输出结构化结果，突出 Top 5 问题与快速修复路径

## 使用示例

- 用户提供 diff 或文件列表后执行评审，返回结构化问题列表与改进建议
- 在合并前进行质量门禁：若存在“严重/高”问题且无修复方案，建议阻塞合并
