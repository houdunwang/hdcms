# Tasks

- [ ] 建立审计框架与报告模板：确定输出结构与字段（严重程度、影响范围、修复建议、定位信息）
- [ ] 扫描项目元信息：识别技术栈、包管理器、测试框架、CI/构建配置
- [ ] 代码规范检查：检测并运行现有 Lint/格式化工具或静态规则，收集违规项
- [ ] 依赖版本与安全审计：解析依赖清单与锁文件，检查兼容性与已知漏洞
- [ ] 单元测试与覆盖率统计：运行测试并生成覆盖率数据，记录关键模块覆盖情况
- [ ] 错误处理机制审查：识别缺失的异常捕获、未处理的 Promise/错误分支、日志与告警合理性
- [ ] 性能与资源效率初步分析：发现潜在热点（大循环/重复 I/O）、构建体积/依赖体积、缓存与数据结构使用
- [ ] 兼容性与环境配置检查：目标运行时/浏览器、Node/语言版本、编译/转译目标与 polyfill
- [ ] 潜在逻辑错误检测：类型检查、不可达/死代码、常见陷阱规则（如空值、边界条件）
- [ ] 汇总与报告生成：将各维度结果汇总为 docs/quality-report.md，按模块排序并给出优先级
- [ ] 结论与改进建议：输出 Top 问题与快速修复路径，附后续提升建议

# Task Dependencies

- “汇总与报告生成” 依赖：审计框架、规范检查、依赖与安全审计、测试覆盖率、错误处理审查、性能分析、兼容性检查、逻辑错误检测
- “单元测试与覆盖率统计” 依赖：扫描项目元信息（以识别测试框架与命令）
- “性能与资源效率初步分析” 依赖：扫描项目元信息（以识别构建/运行方式）
