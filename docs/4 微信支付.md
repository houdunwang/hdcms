# 微信支付

微信支付二维码弹窗组件。点击触发后生成二维码，自动轮询支付状态；支付成功会调用回调并关闭弹窗；二维码倒计时到期后自动刷新。



## 后端配置

首先在 config/hd.ts 中创建支付处理器

* 对象属性 package 是前端支付时要使用的 orderable_type 值

```
payProcess: {
  package: PackageProcess,
},
```

下面是一个处理器的示例

```
import type Order from '#core/models/order';

export default class PackageProcess {
  // 支付成功后，为用户关联课程
  async handle(_order: Order) {
    // 编写业务逻辑，例如：
    // console.log(`为用户 ${order.userId} 开通订阅权限 ${order.orderableId}`)
    return true;
  }

  // 根据课程ID从数据库获取价格
  async getPrice(_id: number): Promise<number> {
    return 0.01
  }
}
```

### Handle 方法

handle方法是在用户支付成功后调用的方法，你在这个方法里处理具体逻辑

* 传递的参数是订单模型对象，你可以mysql数据表中查看字段说明
* order对象的 orderable_type 属性就是上面属性的支付处理器名 package
* order 对象的 orderable_id 属性是前端提交的 orderable_id 值，就是你支付的商品id

### gerPrice 方法

这个方法要根据传递来的id返回出端口的价格，系统自动调用



## 前端使用

```tsx
import { WePay } from '@core/pay/WePay';

export default function Demo() {
  const handleSuccess = () => {
    // 支付成功后的业务逻辑，例如刷新订单状态
  };

  return (
    <WePay
      subject="课程购买"
      orderable_type="package"
      orderable_id={123}
      qrRefreshTime={120}
      onSuccess={handleSuccess}
    >
      支付订单
    </WePay>
  );
}
```

## Props

| 名称 | 类型 | 默认值 | 必填 | 描述 |
| --- | --- | --- | --- | --- |
| subject | string | — | 是 | 支付主题或订单标题 |
| orderable_type | any | — | 是 | 业务类型标识（如订单类型字符串或枚举）访问 admin/config/pay.ts 查看 |
| orderable_id | number | — | 是 | 对应订单或资源的唯一 ID |
| onSuccess | () => void | — | 是 | 支付成功后触发的回调，组件会自动关闭弹窗 |
| children | ReactNode | — | 否 | 显示在二维码上方的自定义内容（如订单摘要） |
| payButton | ReactNode | — | 否 | 自定义触发按钮；不传时显示“微信支付”按钮 |
| qrRefreshTime | number | 120 | 否 | 二维码自动刷新倒计时（秒） |

## 交互说明

- 首次打开弹窗即生成二维码并开始倒计时与支付状态检查。
- 每秒递减倒计时；倒计时为 0 时自动刷新二维码并重置倒计时。
- 支付状态返回 success 时调用 onSuccess 并自动关闭弹窗。
- 触发按钮可自定义；未传入时默认显示“微信支付”按钮。
