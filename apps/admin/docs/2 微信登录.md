1. # 微信扫码登录接口说明

   本文档为开发者提供关于如何集成微信扫码登录功能的详细指引。

   ## 整体开发流程

   前端与后端的完整交互流程如下：
   1. **前端获取二维码**：前端调用后端的“获取登录二维码”接口。
   2. **前端展示并轮询**：前端收到响应后，使用返回的 `qrImg`（Base64 图片数据）来展示二维码，同时使用返回的 `ticket` 开始轮询“使用 Ticket 登录”接口。
   3. **用户扫码确认**：最终用户使用微信扫描二维码并确认登录。
   4. **后端处理回调**：后端服务接收到微信服务器的通知，处理用户信息，并将 `ticket` 与用户 ID 关联起来存入缓存。
   5. **前端登录成功**：正在轮询的“使用 Ticket 登录”接口此时会成功返回 `token` 和用户信息。
   6. **完成集成**：前端获取到 `token` 后，停止轮询，并将 `token` 保存用于后续的用户认证，登录流程结束。

   ***

   ## API 接口详情

   ### 1. 获取登录二维码
   - **作用**
     获取用于微信扫码登录的二维码内容和一个临时票据（`ticket`）。

   - **请求**
     `GET /core/wechat/login/qr`

   - **控制器**
     `LoginController.loginQrCode` (`core/wechat/login_controller.ts`)

   - **参数**
     无

   - **开发者配合**
     1. 调用此接口会得到一个包含 `ticket` 和 `qrImg` 的 JSON 对象。
     2. `qrImg` 字段是一个 Base64 编码的图片数据 URI，您可以直接将它用在 `<img>` 标签的 `src` 属性中来显示二维码。
     3. `ticket` 字段是用于下一步轮询的票据，请务必保存。

   ### 2. 使用 Ticket 登录
   - **作用**
     在用户扫码确认后，使用第一步获取的 `ticket` 来换取用户的登录凭证（`token`）。

   - **请求**
     `POST /core/wechat/login`

   - **控制器**
     `LoginController.login` (`core/wechat/login_controller.ts`)

   - **参数**

   | 参数名   | 类型   | 位置 | 说明                                     |
   | :------- | :----- | :--- | :--------------------------------------- |
   | `ticket` | string | Body | 从“获取登录二维码”接口中得到的临时票据。 |
   - **开发者配合**
     1. 在向用户展示二维码后，立即开始定时（例如每隔1-2秒）轮询调用此接口。
     2. 在用户完成扫码确认之前，此接口的响应会是空的。
     3. 一旦用户确认登录，此接口将返回一个包含 `token` 和 `user` 信息的 JSON 对象。
     4. 接收到 `token` 后，应立即停止轮询，并将 `token` 保存到客户端（如 `localStorage`）用于后续的 API 请求认证。登录完成。
