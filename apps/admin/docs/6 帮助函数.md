# 辅助函数

本文档概述了 `core/helper.ts` 中可用的辅助函数。

---

## `getUserByName(value)`

该异步函数根据提供的标识符从数据库中检索单个用户。它会在 `name`、`email` 和 `mobile` 字段中搜索匹配项。

### 参数

-   `value` (string): 要搜索的值。这可以是用户的姓名、电子邮件地址或手机号码。

### 返回值

-   `Promise<User | null>`: 一个 Promise，它会解析为第一个匹配的 `User` 对象，如果找不到用户，则解析为 `null`。

### 示例

```typescript
import { getUserByName } from '#core/helper'

const user = await getUserByName('john.doe@example.com');
if (user) {
  console.log('找到用户:', user.name);
} else {
  console.log('未找到用户。');
}
```

## `url(as, type)`

该函数为应用程序中的给定路由别名或静态路径生成一个完整的绝对 URL。

**注意:** 当前实现不支持为带参数的路由生成 URL。

### 参数

-   `as` (string): 路由别名 (例如, `'posts.index'`) 或静态路径 (例如, `'/images/avatar.png'`)。
-   `type` ('route' | 'const'): 指定 `as` 参数的类型。
    -   `'route'` (默认): 将 `as` 视为在 AdonisJS 路由器中注册的路由名称。
    -   `'const'`: 将 `as` 视为要附加到应用程序基本 URL 的文本路径。

### 返回值

-   `string`: 完全限定的 URL。

### 示例

**从路由名称生成 URL:**

```typescript
import { url } from '#core/helper'

// 假设 'home' 是为 '/' 注册的路由
const homeUrl = url('home');
// 结果: http://your-app.com/
```

**为静态资源生成 URL:**

```typescript
import { url } from '#core/helper'

const imageUrl = url('/assets/logo.svg', 'const');
// 结果: http://your-app.com/assets/logo.svg
```
