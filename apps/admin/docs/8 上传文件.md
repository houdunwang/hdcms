# 上传文件

## 功能描述

处理文件上传，例如用户头像、媒体文件等。

## 后台配置

文件上传服务的行为由后端的 `.env` 配置文件驱动，允许在不同存储策略之间切换。

- `UPLOAD_DRIVER`: 定义文件存储的方式。
  - `local`: 文件将存储在提供 API 的服务器本地。
  - `oss`: 文件将上传到阿里云对象存储 (OSS)。

如果选择 `oss` 作为驱动，还需要配置以下阿里云相关信息：

- `UPLOAD_OSS_REGION`: 你的 OSS bucket 所在的区域 (e.g., `oss-cn-hangzhou`)。
- `UPLOAD_OSS_BUCKET`: 用于存储文件的 OSS bucket 名称。
- `UPLOAD_OSS_ENDPOINT`: OSS 的访问端点 (e.g., `oss-cn-hangzhou.aliyuncs.com`)。
- `ALIYUN_ACCESS_KEY_ID`: 具有 OSS 操作权限的 AccessKey ID。
- `ALIYUN_ACCESS_KEY_SECRET`: 对应的 AccessKey Secret。

**注意**: 修改这些配置后，可能需要重启后端服务才能生效。

## 接口信息

- **路径**: `/core/upload`
- **方法**: `POST`
- **Content-Type**: `multipart/form-data`

## 请求参数

| 参数名 | 类型   | 描述 | 是否必填 |
| :----- | :----- | :--- | :------- |
| `file` | `file` | 文件 | 是       |

## 响应结果

请求成功后，会返回一个包含文件 URL 的 JSON 对象。

```json
{
  "url": "string"
}
```

## 使用示例

这是一个使用 `@tanstack/react-query` 和 `axios` 在前端实现文件上传的示例。

```typescript
import { useMutation } from '@tanstack/react-query'
import axios from 'axios'

// 定义成功响应的接口
interface UploadResponse {
  url: string
}

// 定义上传文件的异步函数
const uploadFile = async (file: File): Promise<UploadResponse> => {
  // 创建 FormData 对象来包装文件
  const formData = new FormData()
  formData.append('file', file)

  // 发送 POST 请求
  const { data } = await axios.post<UploadResponse>('/core/upload', formData, {
    headers: {
      // 浏览器会自动设置正确的 Content-Type 和 boundary
      'Content-Type': 'multipart/form-data',
    },
  })
  return data
}

/**
 * 用于文件上传的 useMutation hook
 */
export const useUploadFile = () => {
  return useMutation({
    mutationFn: uploadFile,
    onSuccess: (data) => {
      // 上传成功后的回调
      console.log('Upload successful. File URL:', data.url)
      // 在这里可以执行其他操作，例如更新UI、显示通知等
    },
    onError: (error) => {
      // 上传失败后的回调
      console.error('Upload error:', error)
    },
  })
}
```

# 文件上传处理流程

本文档详细说明了项目中的文件上传功能，从接收 HTTP 请求到文件最终存储和数据记录的完整流程。

## 1. 概述

文件上传服务旨在提供一个统一、可配置的文件处理方案。它支持将文件存储在本地服务器或阿里云 OSS，并通过数据库记录每个文件的元数据。该设计确保了无论后端存储如何变化，上传逻辑的核心流程保持一致。

## 2. 核心组件

- **控制器 (`UploadsController`)**: 作为上传功能的入口，负责接收和验证传入的文件请求。
  - 路径: `/Users/hd/code/web/packages/admin/core/controllers/uploads_controller.ts`
- **服务层 (`UploadService`)**: 封装了所有的核心业务逻辑，包括文件移动、与云存储交互以及数据库操作。
  - 路径: `/Users/hd/code/web/packages/admin/core/services/upload_service.ts`
- **配置文件 (`.env`)**: 用于定义上传行为，如选择存储驱动和配置云服务凭证。
  - 路径: `/Users/hd/code/web/packages/admin/.env`

## 3. 配置

上传服务的行为由根目录下的 `.env` 文件中的以下变量控制：

- `UPLOAD_DRIVER`: 指定使用的存储驱动。
  - `local`: 文件将存储在本地服务器。
  - `oss`: 文件将上传到阿里云 OSS。
- `UPLOAD_OSS_REGION`: 阿里云 OSS 的区域。
- `UPLOAD_OSS_BUCKET`: 阿里云 OSS 的存储桶名称。
- `UPLOAD_OSS_ENDPOINT`: 阿里云 OSS 的访问端点。
- `ALIYUN_ACCESS_KEY_ID`: 阿里云 AccessKey ID。
- `ALIYUN_ACCESS_KEY_SECRET`: 阿里云 AccessKey Secret。

**当前配置**: 系统当前配置为使用 `oss` 驱动，将文件上传至阿里云 OSS。

## 4. 处理流程详解

文件上传遵循一个明确的、多阶段的流程：

### 阶段 1: HTTP 请求入口

1.  客户端通过 `multipart/form-data` 格式向服务器发起上传请求。
2.  `UploadsController` 的 `handle` 方法 (`/Users/hd/code/web/packages/admin/core/controllers/uploads_controller.ts:26`) 接收到请求。
3.  控制器从请求中提取文件对象 (`request.file('file')`)。
4.  如果文件存在，控制器调用 `UploadService.upload(file)` 方法，将文件处理委托给服务层。

### 阶段 2: 本地临时存储

- 无论最终存储目标是哪里，`UploadService` 首先会调用 `local(file)` 方法 (`/Users/hd/code/web/packages/admin/core/services/upload_service.ts:125`)。
- 该方法将文件从临时上传目录移动到由 AdonisJS Drive 管理的本地存储区。
- 文件被重命名以避免冲突，路径格式为 `storage/uploads/YYYY/MM/{cuid}.{ext}`。这一步是所有上传策略的共同起点。

### 阶段 3: 驱动策略执行

接下来，服务会检查 `.env` 文件中的 `UPLOAD_DRIVER` 配置，并执行相应的策略。

#### 情况 A: `UPLOAD_DRIVER=oss` (当前配置)

1.  `upload` 方法 (`/Users/hd/code/web/packages/admin/core/services/upload_service.ts:83`) 检测到驱动为 `oss`。
2.  调用 `oss(file)` 方法 (`/Users/hd/code/web/packages/admin/core/services/upload_service.ts:136`) 将文件从本地存储区上传到阿里云 OSS。
    - OSS 中的对象键 (key) 格式为: `YYYY/MM/{userId-or-anonymous}-{cuid}.{ext}`。
    - 上传使用流式传输 (`fs.createReadStream`) 以优化性能。
3.  OSS 上传成功后，服务会从本地磁盘删除该文件的临时副本 (`await disk.delete(file.filePath)`)，以释放空间。
4.  文件的最终 URL 设置为 OSS 返回的 URL。

#### 情况 B: `UPLOAD_DRIVER=local`

1.  如果驱动设置为 `local`，则跳过所有与 OSS 相关的步骤。
2.  文件将保留在阶段 2 中生成的本地路径。
3.  文件的 URL 将是本地服务器上该文件的访问地址。

### 阶段 4: 数据库记录

1.  在文件成功存储（无论是本地还是 OSS）并获得最终 URL 后，`upload` 方法会调用 `saveToDatabase(file, url)` (`/Users/hd/code/web/packages/admin/core/services/upload_service.ts:105`)。
2.  此方法会在 `uploads` 数据库表中创建一个新记录，包含以下信息：
    - `url`: 文件的最终可访问 URL。
    - `name`: 文件的原始客户端名称。
    - `userId`: 上传用户的 ID (如果用户已登录)。
    - `driver`: 使用的存储驱动 (`local` 或 `oss`)。
    - `size`: 文件大小（字节）。
    - `extension`: 文件扩展名。
3.  `UploadService.upload` 方法最终返回这个新创建的 `Upload` 模型实例。
