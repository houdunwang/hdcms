# 微信账号绑定流程

本文档旨在帮助开发者理解并使用微信账号绑定功能。

## 1. 流程介绍

微信绑定功能允许用户将他们的系统账户与微信 OpenID 关联起来，以便后续通过微信进行登录或其他操作。

整体流程如下：

1.  **获取二维码**：用户在需要绑定微信的页面（如个人中心）点击“绑定”按钮。前端应用向后端服务发起请求，获取一个专用于本次绑定流程的微信二维码。
2.  **用户扫码**：前端将获取到的二维码展示给用户。
3.  **微信确认**：用户使用手机微信扫描该二维码，并在手机上点击确认。
4.  **后台处理**：
    - 用户确认后，微信服务器会向我们在后台配置的回调地址发送一条事件消息。
    - 后台服务接收到该消息，消息中包含了用户的 `openid` 和一个临时的 `ticket`。
    - 服务会将 `openid` 以 `ticket` 为键，存入缓存中（有效期为1分钟）。同时，通过微信回复用户“绑定成功”。
5.  **完成绑定**：前端在展示二维码后，会通过轮询或长连接等方式等待用户扫码。扫码成功后，前端会得到通知，然后携带从第一步获取的 `ticket` 向后端发起最终的绑定请求。
6.  **数据关联**：后端收到绑定请求后，会验证 `ticket` 的有效性，从缓存中取出 `openid`，并将其与当前登录用户的账户信息进行关联并保存到数据库。
7.  **绑定成功**：后端向前端返回成功消息，前端更新UI，提示用户绑定成功。

## 2. 接口说明

### 2.1 获取绑定二维码

此接口用于生成并获取一个用于微信扫码绑定的二维码。

- **URL**：`/core/wechat/bind/qr`
- **Method**：`GET`
- **Controller**：`core/wechat/bind_controller.ts` -> `loginQrCode`
- **所需权限**：用户必须处于登录状态。
- **请求参数**：无
- **成功响应示例**：

  ```json
  {
    "ticket": "gQG28DoAAAAAAAAAASxodHRwOi8vd2VpeGluLnFxLmNvbS9xL0JqZ2c4PXJtZ2c3eWJ0b3BfAAIEZ23pXwMEAAAAAA==",
    "expire_seconds": 60,
    "url": "http://weixin.qq.com/q/Bjg_8=rmgg7ybtop_",
    "qrImg": "data:image/png;base64,iVBORw0KGgoAAAANSUhEUgAA..."
  }
  ```

  **字段说明**:
  - `ticket`: 获取二维码的凭证，用于换取二维码图片，并在后续的绑定请求中使用。
  - `expire_seconds`: 二维码的有效时间，以秒为单位。
  - `url`: 二维码解析后的地址，开发者可根据此地址自行生成需要的二维码。
  - `qrImg`: Base64 编码的二维码图片，可直接在 `<img>` 标签的 `src` 属性中使用。

### 2.2 执行绑定

前端在获取到二维码后，应采用轮询（Polling）机制，**每隔一段时间（例如：2秒）** 调用此接口，以检查用户是否已完成扫码操作。

- **URL**：`/core/wechat/bind`
- **Method**：`POST`
- **Controller**：`core/wechat/bind_controller.ts` -> `bind`
- **所需权限**：用户必须处于登录状态。
- **请求参数**：

| 参数名   | 类型   | 是否必须 | 描述                                        |
| -------- | ------ | -------- | ------------------------------------------- |
| `ticket` | string | 是       | 调用“获取绑定二维码”接口时返回的 `ticket`。 |

- **响应示例**：

  后端会根据扫码状态返回不同的响应。前端应根据响应体中的 `data.status` 字段来判断后续操作。
  - **等待扫码** (`status: 'waiting'`)

    当用户还未扫码，或 `ticket` 已过期时，会返回此状态。前端应继续轮询，直到状态变为 `success` 或轮询超时。

    ```json
    {
      "status": "success",
      "message": "等待扫码",
      "data": {
        "status": "waiting"
      }
    }
    ```

  - **绑定成功** (`status: 'success'`)

    当用户成功扫码后，后端完成绑定操作，并返回此状态。前端应停止轮询，并给出绑定成功的提示。

    ```json
    {
      "status": "success",
      "message": "绑定成功",
      "data": {
        "status": "success"
      }
    }
    ```
